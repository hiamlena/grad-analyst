# Input Sufficiency Contract — ГрадоАналитик

## Обязательные входные данные

Пользователь должен предоставить:

1. Кадастровый номер.
2. Адрес участка.
3. Площадь участка.
4. Планируемый тип объекта (что хочет построить).
5. Документы (по возможности): ПЗЗ, ГПЗУ, ЕГРН.

## Принцип ответа

ИИ не должен делать выводы «из головы».

Если данных недостаточно, ответ должен содержать формулировку:

**«Данных недостаточно. Для вывода нужно загрузить: ...»**

где вместо `...` перечисляются конкретно недостающие данные/документы.

## Минимальный формат результата

- Статус: `можно` / `нельзя` / `возможно при условиях` / `данных недостаточно`.
- Краткое обоснование только по предоставленным данным.
- Список недостающих данных (если применимо).
