# UI-спецификация MVP: первый экран «ГрадоАналитик»

## Назначение экрана
Первый экран сервиса предназначен для сбора исходных данных по земельному участку, выбора цели анализа, загрузки документов и запуска предварительного ИИ-анализа с выводом результата и базового 2D-эскиза.

## 1) Шапка
Состав шапки:
- Название сервиса: **«ГрадоАналитик»**.
- Подпись (подзаголовок): **«Предварительный ИИ-анализ земельного участка»**.
- Предупреждение: **«Не является юридическим заключением»**.

Требование к акцентам:
- Название — главный визуальный акцент.
- Предупреждение — заметное, но вторичное по иерархии относительно названия.

## 2) Блок «Данные участка»
В блоке должны быть поля:
- Кадастровый номер.
- Адрес.
- Площадь участка.
- Размеры участка (если известны).
- Сторона подъезда.

Требования к UX:
- Поля должны поддерживать ручной ввод.
- Для неизвестных параметров (например, размеры) допустимо оставить пустым.

## 3) Блок «Цель анализа»
В блоке предусмотрены:
- Режим: **«Я знаю, что хочу построить»**.
- Режим: **«Подобрать лучший вариант»**.
- Поле: **«Что планируется построить»**.

Логика отображения:
- Пользователь выбирает один из двух режимов.
- Поле «Что планируется построить» доступно в обоих режимах, но в режиме подбора может быть необязательным.

## 4) Блок «Документы»
Должны быть зоны/контролы загрузки для:
- ПЗЗ.
- ГПЗУ.
- ЕГРН.
- Схемы участка.
- Дополнительных файлов.

Требования:
- Разрешить по одному или несколько файлов в «Дополнительные файлы».
- Показывать список загруженных файлов внутри блока.

## 5) Кнопка запуска
Кнопка основного действия:
- **«Сформировать предварительный анализ»**.

Требования:
- Кнопка визуально выделена как primary action.
- До ввода минимально необходимых данных кнопка может быть неактивной.

## 6) Блок результата
После запуска анализа отображается результат со следующими элементами:
- Статус: **можно / нельзя / возможно при условиях / данных недостаточно**.
- Краткий вывод.
- Риски.
- Рекомендации.
- Что нужно дозагрузить.
- Источники.

Требования к представлению:
- Статус должен быть максимально заметен и считываться первым.
- Разделы «Риски» и «Рекомендации» отображаются списками.
- «Источники» — перечень документов/нормативных оснований, использованных в выводе.

## 7) Блок 2D-эскиза
Визуальный блок эскиза должен включать:
- Прямоугольник участка.
- Отступы.
- Пятно застройки.
- Объект.
- Въезд.
- Подписи.

Требования к эскизу:
- Эскиз носит предварительный иллюстративный характер.
- Подписи должны однозначно идентифицировать элементы (границы, отступы, въезд, объект).

## Ограничения реализации в рамках MVP-спецификации
- Только описание визуала и структуры экрана.
- Без frontend-кода.
- Без создания Next.js-проекта.
- Без интеграции OpenAI.
- Без подключения базы данных.
