# Input Sufficiency Contract — ГрадоАналитик

## Область MVP

MVP рассчитан на одного пользователя и выполняет предварительный анализ без подключения Росреестра, карт, GIS, автоматического определения территориальной зоны, авторизации или внешних интеграций.

ПЗЗ, градостроительные регламенты и справочные документы могут храниться на сервере как внутренняя база знаний сервиса. Пользователь не обязан каждый раз загружать ПЗЗ, если подходящий документ уже есть в серверной базе сервиса.

## Обязательные входные данные пользователя

Пользователь должен предоставить через форму:

1. Кадастровый номер.
2. Адрес участка / муниципалитет.
3. Площадь участка.
4. Что планируется построить.
5. Размеры участка, если известны.
6. Сторона подъезда.

## Серверные источники сервиса

Сервис может использовать внутреннюю серверную базу документов, если подходящие документы уже добавлены администратором/оператором сервиса:

1. ПЗЗ.
2. Градостроительные регламенты.
3. Карту градостроительного зонирования, если она добавлена.
4. Местные нормативы.
5. Справочные документы.

## Документы пользователя

Пользователь может загрузить:

1. ГПЗУ.
2. ЕГРН.
3. Схему участка.
4. Топосъёмку.
5. Документы по ЗОУИТ.
6. Дополнительные документы.
7. ПЗЗ или фрагмент карты градостроительного зонирования, если у пользователя есть более актуальная версия.

## Разрешённые источники для анализа

ИИ может использовать только:

- документы из серверной базы сервиса;
- документы, загруженные пользователем;
- данные формы.

ИИ не должен использовать внешние источники, Росреестр, карты, GIS, автоматическое определение зоны или знания «из головы».

## Принцип ответа

ИИ не должен делать выводы без документального основания.

Если по адресу или кадастровому номеру нельзя определить нужный ПЗЗ или территориальную зону на основании серверных источников, пользовательских документов или данных формы, ответ должен содержать формулировку:

**«Данных недостаточно. Для вывода нужно загрузить ГПЗУ или фрагмент карты градостроительного зонирования / указать территориальную зону.»**

В остальных случаях, если данных недостаточно по другой причине, ответ должен содержать формулировку:

**«Данных недостаточно. Для вывода нужно загрузить: ...»**

где вместо `...` перечисляются конкретно недостающие данные/документы.

## Минимальный формат результата

- Статус: `можно` / `нельзя` / `возможно при условиях` / `данных недостаточно`.
- Краткое обоснование только по разрешённым источникам.
- Раздел источников с разделением на:
  - источники сервиса;
  - документы пользователя.
- Список недостающих данных (если применимо).
